Set与WeakSet

Set

简述

Set对象是值的集合，可以储存任何类型，与数组不同的是，Set中相同的值只能出现一次。

属性和方法

属性

  • size：返回集合所包含元素的数量

方法

  • 操作方法
    • add(value)：向集合添加一个新的项
    • delete(value)：从集合中移除一个值
    • has(value)：如果值在集合中存在，返回true,否则false
    • clear(): 移除集合里所有的项
  • 遍历方法
    • keys()：返回一个包含集合中所有键的迭代器对象
    • values()：返回一个包含集合中所有值的迭代器对象
    • entries()：返回一个包含集合中所有键值对的迭代器对象
    • forEach()：用于对集合成员执行遍历，没有返回值

Tips

NaN和undefined都可以被存储在Set 中， NaN之间被视为相同的值（NaN被认为是相同的，尽管 NaN !== NaN）。

Set内部使用Object.is判断两个值是否相等，对于+0与-0则存在例外情况，Set对象会把这两个值视为相等。

WeakSet

简述

WeakSet用于储存弱引用对象，如果我们将其用于储存基本类型值，将会执行出错。

当WeakSet 存储对象时，垃圾回收机制将不考虑 WeakSet 对该对象的引用，也就是说，如果其他对象都不再引用该对象，那么垃圾回收机制会自动回收该对象所占用的内存，不考虑该对象还存在于 WeakSet 之中。

需要注意的是，我们无法知晓何时进行垃圾回收，在浏览器环境执行相关代码时，所以我们有时候可能会发现WeakSet仍然存在应当被销毁的对象。

属性和方法

由于WeakMap存储的是弱引用对象，所以我们无法确切的的得知其中的内容，所以与其相关的size属性、clear方法以及所有的遍历方法全部都不具有，在使用中需要与Set做出区分。

应用场景

我们可以用WeakMap弱引用特性去存储页面上的可变列表，这样我们也不用担心会造成内存泄漏。

总结

在日常工作学习当中我们很少用到Set和WeakSet。不过它们的一些特性还是很有价值的，会在很多项目的源码中被使用到，我们了解这些特性可以能够更好的去学习源码。
