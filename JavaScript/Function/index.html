声明方式

在浏览器环境中，func3函数会被挂载到window对象上，会造成全局变量污染，因此不建议在浏览器环境当中使用这种方式声明函数，在node环境下不存在该问题。

匿名函数与函数提升

我们称上述声明方式中func2为匿名函数，func3为具名函数。这两种方式声明有一些差异，具名函数声明会在代码执行开始时被提升至最顶部，而匿名函数则不会。具体可看代码。

立即执行函数

有时候需要对外界隐藏自身的一些API，不想污染全局变量，这个时候就可以用到立即执行函数去创建一块新的块级作用域去保存我们的变量。

关于立即执行函数还有一些其他用途，我们在之后的章节会讲到它。

递归函数

通过在函数内部条用函数本身的方式实现一些比较复杂的功能，比如阶乘运算。

回调函数

在函数内部调用传递过来的函数参数，我们称这个函数参数为回调函数。

默认参数与剩余参数

在传递参数时可以对参数设置默认值，也可以收集剩余的一些参数。

箭头函数

箭头函数是ES6中新增的一种函数的简化式声明方法，可以让我们少些更多的代码。

箭头函数在带来了代码优化的同时，它也改变了函数内部this的调用指向，我们需要进行注意。有关函数this指向在下面将会讲到。

函数与方法内部this指向

尤其要注意到的是，在方法内部的函数中，this指向会指向全局。

箭头函数带来的this指向改变

箭头函数不会创建自己的this,它只会从自己的作用域链的上一层继承this。我们可以看一下这个例子就能明白是怎么回事了。

cal, apply, bind

call与apply在使用时会调用函数并直接改变函数内部的this指向

我们可以利用这种特性实现对象的继承。

bind与上述两种区别在于不立刻执行。。
